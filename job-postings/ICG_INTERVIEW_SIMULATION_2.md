- Identify the specific point of divergence

**Step 2: Data Validation**
- Pull raw data samples for both systems
- Manually calculate closure rate using both methodologies
- Check for data refresh timing differences (real-time vs. batch updates)
- Look for filtering differences (date ranges, status definitions, exclusions)
- Verify data source alignment (are we querying the same systems?)

- Most common issues I've seen:
  - **Definition differences**: \"Closed\" might mean different statuses

  - **Filter misalignment**: Dashboard excludes certain categories they include

**Step 4: Solution & Validation**
- Align on single source of truth for closure rate definition
- Update dashboard logic if needed
- Run parallel tracking for 1 week to validate alignment
- Document the agreed-upon methodology for future reference

**Step 5: Prevention**
- Create data dictionary defining all metrics
- Set up automated reconciliation checks between systems
- Schedule regular metric alignment meetings

This is exactly the kind of systematic debugging I did when reconciling data accuracy metrics across 3 client accounts at Ausbiz."
